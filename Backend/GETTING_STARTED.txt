STEP 1: Add Rate Limiting Middleware to Server.js
─────────────────────────────────────────────────

Find this in your Server.js:
    app.use(verifyToken);     // Authentication middleware

Add these lines right after:
    const { rateLimitMiddleware } = require('./Middleware/rateLimiter');
    const rateLimitRoutes = require('./Routes/RateLimitRoutes');
    
    app.use(rateLimitMiddleware);
    app.use('/api/rate-limit', rateLimitRoutes);

Then add all your existing routes...


STEP 2: Test it Works
─────────────────────

Terminal 1 - Start backend:
    npm start

Terminal 2 - Test rate limiting (make 150 requests):
    for i in {1..150}; do curl http://localhost:5000/api/students & done; wait

Expected: Some requests succeed (200), others get "429 Too Many Requests"


STEP 3: Deploy to Production
─────────────────────────────

Install PM2 globally:
    npm install -g pm2

Start with PM2:
    pm2 start ecosystem.config.js

View logs:
    pm2 logs Backend

Done! ✅

Issue: "rateLimitMiddleware is not a function"
Fix:   Check the import path in Server.js
       Should be: './Middleware/rateLimiter'

Issue: All requests getting 429
Fix:   Make sure authentication middleware is before rate limiter
       Rate limiter needs req.user._id to identify users

Issue: Server crashes
Fix:   Use PM2: pm2 start ecosystem.config.js
       Provides memory protection and auto-restart

Issue: High memory usage
Fix:   Restart server: pm2 restart ecosystem.config.js
       Or use: node --max-old-space-size=2048 Server.js

Issue: Socket.io connections dropping
Fix:   Check if MAX_CONNECTIONS reached
       Monitor with: pm2 logs Backend
